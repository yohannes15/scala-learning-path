# Static assets (diagrams)

## Why this is useful

Some concepts are easier to **see** than to read in prose: the **Scala type lattice** (`Any` → `AnyVal` / `AnyRef` → `Nothing`), **numeric widening**, and the **collections** trait hierarchy. These SVGs support the written tutorials and are linked from the main docs.

## What’s covered

| Asset | Content |
|-------|---------|
| `type-hierarchy.svg` | Core types: `Any`, `Matchable`, `AnyVal`, `AnyRef`, `Nothing`, `Null` |
| `type-casting-diagram.svg` | Widening between numeric types and `Char` |
| `scala.collection-high-level.svg` | High-level collection traits (`Iterable`, `Seq`, `Set`, `Map`, …) |
| `scala.collection.immutable.svg` | Default **immutable** concrete types |
| `scala.collection.mutable.svg` | **Mutable** counterparts |

**Rendered index:** see [`docs/diagrams.md`](../../../../../docs/diagrams.md) in the repository root for embedded previews and tables.
